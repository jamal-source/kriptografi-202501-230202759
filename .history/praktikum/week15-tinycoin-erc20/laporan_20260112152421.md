## 2. Dasar Teori
ERC20 adalah standar token untuk Ethereum yang mendefinisikan aturan untuk token yang dapat dipertukarkan. Standar ini mencakup fungsi-fungsi dasar seperti transfer, approve, dan transferFrom. TinyCoin adalah implementasi sederhana dari token ERC20 yang menggunakan library OpenZeppelin untuk memastikan keamanan dan kepatuhan standar.

Smart contract TinyCoin menggunakan Solidity versi 0.8.0 atau lebih baru, yang sudah memiliki built-in protection terhadap overflow/underflow. Kontrak ini mewarisi dari ERC20 dan mengimplementasikan fungsi minting awal di constructor.

## 5. Source Code
```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "@openzeppelin/contracts/token/ERC20/ERC20.sol";

contract TinyCoin is ERC20 {
    constructor(uint256 initialSupply) ERC20("TinyCoin", "TNC") {
        _mint(msg.sender, initialSupply);
    }
}
```

### Analisis Keamanan
Kontrak ini menggunakan OpenZeppelin ERC20 yang sudah teruji keamanannya. Risiko potensial:
- Reentrancy: Tidak berlaku karena tidak ada external call
- Overflow: Solidity 0.8+ sudah mencegah overflow
- Access control: Constructor mint hanya untuk deployer

## 7. Jawaban Pertanyaan
- Pertanyaan 1: Apa fungsi utama ERC20 dalam ekosistem blockchain?
  ERC20 adalah standar untuk token yang dapat dipertukarkan di Ethereum. Fungsi utamanya adalah memungkinkan transfer token antar alamat, approve spending oleh pihak ketiga, dan query saldo/total supply. Ini memungkinkan interoperabilitas antara berbagai aplikasi DeFi, exchange, dan wallet.

- Pertanyaan 2: Bagaimana mekanisme transfer token bekerja dalam kontrak ERC20?
  Transfer dilakukan melalui fungsi transfer(to, amount) yang mengurangi saldo sender dan menambah saldo receiver. Fungsi ini memverifikasi bahwa sender memiliki saldo cukup sebelum melakukan transfer. Event Transfer di-emit untuk mencatat transaksi.

- Pertanyaan 3: Apa risiko utama dalam implementasi smart contract dan bagaimana cara mitigasinya?
  Risiko utama: reentrancy attacks, integer overflow/underflow, access control issues. Mitigasi: gunakan library teruji seperti OpenZeppelin, enable compiler checks, lakukan code review, dan gunakan tools seperti Slither untuk analisis keamanan.
