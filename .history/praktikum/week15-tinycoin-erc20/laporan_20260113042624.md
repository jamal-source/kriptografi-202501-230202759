## 2. Dasar Teori
ERC20 adalah standar token untuk Ethereum yang mendefinisikan aturan untuk token yang dapat dipertukarkan. Standar ini mencakup fungsi-fungsi dasar seperti transfer, approve, dan transferFrom. TinyCoin adalah implementasi sederhana dari token ERC20 yang menggunakan library OpenZeppelin untuk memastikan keamanan dan kepatuhan standar.

Smart contract TinyCoin menggunakan Solidity versi 0.8.0 atau lebih baru, yang sudah memiliki built-in protection terhadap overflow/underflow. Kontrak ini mewarisi dari ERC20 dan mengimplementasikan fungsi minting awal di constructor.

### Cara Menjalankan Kontrak TinyCoin
Karena Solidity tidak dapat dijalankan langsung seperti Python, kontrak ini perlu di-deploy menggunakan Remix IDE:

1. Buka [Remix IDE](https://remix.ethereum.org/) di browser.
2. Buat file baru `TinyCoin.sol` dan copy kode kontrak.
3. Pada tab "Solidity Compiler", pilih versi ^0.8.0 dan klik "Compile".
4. Pada tab "Deploy & Run Transactions", pilih environment "JavaScript VM".
5. Masukkan initial supply (contoh: 1000000) dan klik "Deploy".
6. Setelah deploy, gunakan fungsi balanceOf, transfer, dll untuk testing.
7. Ambil screenshot hasil deployment dan transaksi.

## 5. Source Code
```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;


import "@openzeppelin/contracts@4.9.6/token/ERC20/ERC20.sol";
import "@openzeppelin/contracts@4.9.6/access/Ownable.sol";

contract TinyCoin is ERC20, Ownable {

    constructor() ERC20("TinyCoin", "TNC") {
        // Initial supply: 1,000,000 TNC
        _mint(msg.sender, 1_000_000 * 10 ** decimals());
    }

    // Mint token (only owner)
    function mint(address to, uint256 amount) external onlyOwner {
        _mint(to, amount);
    }

    // Burn token (by holder)
    function burn(uint256 amount) external {
        _burn(msg.sender, amount);
    }
}

```

7. Jawaban Pertanyaan

1. Apa fungsi utama ERC20 dalam ekosistem blockchain?
ERC20 berfungsi sebagai standar token agar dapat digunakan secara universal oleh wallet, exchange, dan aplikasi DeFi.

2. Bagaimana mekanisme transfer token bekerja?
Fungsi transfer() memindahkan token dari pengirim ke penerima dengan memastikan saldo mencukupi, lalu mencatat transaksi melalui event Transfer.

3. Apa risiko utama smart contract dan mitigasinya?
Risiko utama meliputi reentrancy, overflow, dan kesalahan kontrol akses. Mitigasi dilakukan dengan menggunakan library OpenZeppelin dan compiler Solidity versi terbaru.
