# Laporan Penuh Proyek KoinAllIn ERC-20 dengan DApp MetaMask

## Pendahuluan
Proyek KoinAllIn adalah implementasi lengkap token ERC-20 dengan fitur tambahan burning dan staking menggunakan bahasa pemrograman Solidity, dilengkapi dengan Decentralized Application (DApp) berbasis web yang terintegrasi dengan MetaMask wallet. Proyek menggunakan alat bantu seperti Brownie, Web3.js, dan Infura untuk pengembangan, pengujian, dan penyebaran ke jaringan utama Ethereum. Token ini dinamakan KoinAllIn (AIC) dengan pasokan awal 1.000.000 AIC. Proyek ini telah berhasil di-deploy ke Ethereum mainnet dengan DApp yang fully functional, menjadikan koin ini "nyata" dan dapat diperdagangkan serta diinteraksikan melalui interface web modern.

## Apa yang Telah Kita Lakukan Sebelumnya
Dalam pengembangan proyek KoinAllIn, kami telah menyelesaikan beberapa langkah penting:
- Membuat struktur direktori proyek: kontrak, skrip, build.
- Mengubah nama file TinyToken.sol menjadi TinyCoin.sol di folder kontrak.
- Membuat file konfigurasi brownie-config.yaml.
- Mengembangkan skrip deploy.py untuk penyebaran kontrak.
- Mengembangkan skrip interact.py untuk interaksi dengan kontrak, termasuk transfer token dan pengecekan saldo.
- Mengatasi masalah instalasi, seperti instalasi Microsoft Visual C++ Build Tools untuk mengatasi error cchecksum.
- Memindahkan proyek ke drive D untuk mengatasi masalah ruang disk.
- Membuat lingkungan virtual di WSL untuk instalasi dependensi Python.
- Menginstal dependensi Python seperti eth-brownie dan web3.
- Menginstal Ganache untuk simulasi blockchain lokal.
- Menginstal dependensi Brownie.
- Membuat dokumentasi README.md dengan instruksi penggunaan.

## Kekurangan
- Tidak ada fitur tambahan seperti burning token atau minting dinamis.
- Belum ada pengujian unit yang komprehensif; hanya interaksi dasar yang diuji.
- Dependensi pada alat eksternal seperti Brownie dan Infura, yang mungkin sulit diinstal di beberapa sistem.
- Masih memerlukan verifikasi kontrak di Etherscan untuk transparansi penuh.

## Kelebihan
- Menggunakan standar ERC-20 yang terbukti, memastikan kompatibilitas dengan dompet dan pertukaran yang mendukung ERC-20.
- Penggunaan Brownie memudahkan penyebaran dan interaksi dengan kontrak pintar.
- Struktur proyek yang terorganisir dengan pemisahan antara kontrak, skrip, dan build.
- Pasokan awal yang terbatas (1000 TNC) membuat token ini cocok untuk proyek kecil atau demonstrasi.
- Kode Solidity yang sederhana dan mudah dipahami, dengan implementasi fungsi transfer, approve, dan transferFrom.

## Kendala
- Sistem operasi Windows memerlukan instalasi tambahan seperti Microsoft Visual C++ Build Tools, yang bisa rumit.
- Keterbatasan ruang disk di drive utama memaksa pemindahan proyek.
- Dependensi pada Python 3.8+, Node.js, dan npm, yang harus diinstal terlebih dahulu.
- Penggunaan WSL untuk lingkungan virtual, yang mungkin tidak familiar bagi pengguna Windows biasa.
- Konfigurasi solc compiler yang spesifik, termasuk path ke executable solc.

## Langkah-Langkah Pembuatan dan Progres
1. **Persiapan Lingkungan**: Instal Python 3.8+, Node.js, npm, dan Microsoft Visual C++ Build Tools.
2. **Struktur Proyek**: Buat direktori kontrak, skrip, build, dan file konfigurasi.
3. **Pengembangan Kontrak**: Tulis kontrak KoinAllIn.sol dengan fungsi ERC-20 plus burning dan staking.
4. **Skrip Penyebaran**: Buat deploy_patched.py untuk penyebaran kontrak dengan pasokan awal dan MetaMask integration.
5. **Skrip Interaksi**: Buat interaksi.py untuk transfer token dan pengecekan saldo.
6. **Konfigurasi Brownie**: Atur brownie-config.yaml dengan dependensi dan pengaturan compiler.
7. **Instalasi Dependensi**: Instal eth-brownie, web3, Ganache, dan dependensi Brownie.
8. **Pengujian**: Jalankan Ganache, sebar kontrak, dan lakukan interaksi.
9. **Dokumentasi**: Buat README.md dan TODO.md untuk melacak progres.

## Cara Menjalankan
1. **Instalasi Prasyarat**:
   - Python 3.8+
   - Node.js dan npm
   - Microsoft Visual C++ 14.0 atau lebih tinggi

2. **Instalasi Dependensi**:
   ```bash
   pip install eth-brownie web3
   ```
   Atau di lingkungan virtual:
   ```bash
   python -m venv venv
   venv\Scripts\activate
   pip install eth-brownie web3
   ```

3. **Instal Ganache**:
   ```bash
   npm install -g ganache
   ```

4. **Instal Dependensi Brownie**:
   ```bash
   brownie pm install
   ```

5. **Jalankan Ganache**:
   ```bash
   ganache
   ```

6. **Sebar Kontrak**:
   ```bash
   brownie run skrip/deploy_patched.py --network mainnet
   ```

7. **Interaksi dengan Kontrak**:
   ```bash
   brownie run skrip/interaksi.py --network mainnet
   ```

## Pengembangan DApp MetaMask

### Fitur DApp Utama
Proyek KoinAllIn telah dikembangkan menjadi Decentralized Application (DApp) lengkap dengan fitur-fitur berikut:

#### 1. **Integrasi MetaMask**
- Koneksi otomatis dengan MetaMask wallet
- Deteksi presence MetaMask di browser
- Fallback ke Infura untuk operasi read-only jika MetaMask tidak tersedia
- Request account access dengan user permission

#### 2. **Fungsi Token Lengkap**
- **Check Balance**: Melihat saldo AIC dengan real-time updates
- **Transfer Tokens**: Transfer AIC ke address lain dengan validasi input
- **Burn Tokens**: Membakar token untuk mengurangi supply
- **Stake Tokens**: Menambahkan token ke staking pool
- **Check Stake**: Melihat jumlah token yang di-stake
- **Withdraw Stake**: Menarik token dari staking pool
- **Check Rewards**: Melihat reward yang diperoleh dari staking

#### 3. **Error Handling & Validation**
- Validasi address Ethereum format
- Validasi input amount (tidak boleh negatif/kosong)
- Deteksi requirement MetaMask untuk transaksi
- Pesan error yang informatif dan user-friendly
- Status feedback real-time untuk semua operasi

#### 4. **User Interface**
- Design responsif untuk desktop dan mobile
- Status indicators dengan warna (hijau untuk success, merah untuk error)
- Real-time balance updates setelah transaksi
- Loading states dan progress feedback

### Teknologi DApp
- **Frontend**: HTML5, CSS3, JavaScript ES6+
- **Blockchain Integration**: Web3.js 1.8.0
- **Wallet**: MetaMask extension
- **RPC Provider**: Infura Ethereum mainnet
- **Styling**: CSS modern dengan responsive design

### Cara Menggunakan DApp
1. **Setup**: Install MetaMask extension di browser
2. **Connect**: Klik "Connect MetaMask" untuk menghubungkan wallet
3. **Interact**: Gunakan berbagai fungsi token melalui interface web
4. **Monitor**: Pantau balance dan status transaksi secara real-time

### Keamanan DApp
- Semua transaksi memerlukan konfirmasi MetaMask
- Private keys tetap aman di MetaMask wallet
- Tidak ada penyimpanan data sensitif di frontend
- Validasi input di client-side dan contract-side
